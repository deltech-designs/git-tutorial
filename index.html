Git Tutorial

Welcome to the Git tutorial! This guide will help you understand the basics of using Git for version control.

Getting Started

To start using Git, you need to install it on your machine. You can download it from the official website: Git Downloads.

Basic Commands

  • git init - Initialize a new Git repository.
  • git clone [url] - Clone an existing repository from a remote server.
  • git add [file] - Stage changes to be committed.
  • git commit -m "message" - Commit staged changes with a message.
  • git push - Push changes to the remote repository.
  • git pull - Fetch and merge changes from the remote repository.
  • git status - Check the status of your repository.
  • git log - View the commit history.
  • git branch - List, create, or delete branches.
  • git checkout [branch] - Switch to a different branch.
  • git merge [branch] - Merge changes from one branch into another.
  • git reset [file] - Unstage a file that has been staged.
  • git rm [file] - Remove a file from the working directory and stage the removal.
  • git stash - Stash changes in a dirty working directory.
  • git tag [name] - Create a tag for a specific commit.
  • git fetch - Download objects and refs from another repository.
  • git remote add [name] [url] - Add a new remote repository.
  • git remote -v - List all remote repositories.
  • git config --global user.name "Your Name" - Set your Git username.
  • git config --global user.email "your.email@example.com" - Set your Git email address.
  • git config --list - List all Git configuration settings.
  • git help [command] - Get help for a specific Git command.
  • git bisect - Use binary search to find a commit that introduced a bug.
  • git cherry-pick [commit] - Apply changes from a specific commit to the current branch.
  • git rebase [branch] - Reapply commits on top of another base tip.
  • git revert [commit] - Create a new commit that undoes the changes made by a previous commit.
  • git show [commit] - Show the changes made in a specific commit.
  • git diff - Show changes between commits, commit and working tree, etc.
  • git archive - Create an archive of files from a named tree.
  • git blame [file] - Show what revision and author last modified each line of a file.
  • git reflog - Show the reference logs for the repository.
  • git gc - Cleanup unnecessary files and optimize the local repository.
  • git submodule - Manage external repositories within a repository.
  • git worktree - Manage multiple working trees attached to the same repository.
  • git config --global alias.[alias] [command] - Create a shortcut for a Git command.
  • git ls-files - Show information about files in the index.
  • git ls-remote [url] - List references in a remote repository.
  • git clean -f - Remove untracked files from the working directory.
  • git cherry - Find commits that are in one branch but not in another.
  • git describe - Give an object a human-readable name based on the nearest tag.
  • git grep [pattern] - Search for a pattern in the working directory.
  • git notes - Add, remove, or list notes attached to objects.
  • git replace [old] [new] - Replace an object with another object.
  • git shortlog - Summarize git log output.
  • git tag -d [tag] - Delete a tag from the repository.
  • git push origin --delete [branch] - Delete a branch from the remote repository.
  • git pull --rebase - Fetch and reapply changes from the remote repository.
  • git config --global core.editor [editor] - Set the default text editor for Git.
  • git config --global merge.tool [tool] - Set the default merge tool for Git.
  • git config --global diff.tool [tool] - Set the default diff tool for Git.
  • git config --global alias.st status - Create an alias for the status command.
  • git config --global alias.co checkout - Create an alias for the checkout command.
  • git config --global alias.br branch - Create an alias for the branch command.
  • git config --global alias.cm commit - Create an alias for the commit command.
  • git config --global alias.lg "log --oneline --graph" - Create an alias for a concise log view.
  • git config --global alias.last "log -1 HEAD" - Create an alias to view the last commit.
  • git config --global alias.unstage "reset HEAD --" - Create an alias to unstage files.
  • git config --global alias.visual "log --graph --oneline" - Create an alias for a visual log.
  • git config --global alias.hist "log --pretty=format:'%h %ad | %s%d [%an]' --graph --date=short" - Create an alias for a detailed history log.
  • git config --global alias.alias "config --get-regexp alias" - Create an alias to list all aliases.
  • git config --global alias.fixup "commit --fixup" - Create an alias to create a fixup commit.
  • git config --global alias.amend "commit --amend" - Create an alias to amend the last commit.
  • git config --global alias.rebasei "rebase -i" - Create an alias for interactive rebase.
  • git config --global alias.stashp "stash pop" - Create an alias to pop the last stash.
  • git config --global alias.stashl "stash list" - Create an alias to list stashes.
  • git config --global alias.stashc "stash create" - Create an alias to create a stash.
  • git config --global alias.stashd "stash drop" - Create an alias to drop a stash.
  • git config --global alias.stashs "stash show" - Create an alias to show the last stash.
  • git config --global alias.stashc "stash clear" - Create an alias to clear all stashes.
  • git config --global alias.stashs "stash save" - Create an alias to save a stash.
  • git config --global alias.stashp "stash pop" - Create an alias to pop the last stash.
  • git config --global alias.stashl "stash list" - Create an alias to list all stashes.
